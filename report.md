# Описание подхода

## Задача

Бинарная сегментация разломов в 3D сейсмических объёмах. На вход подаётся объём 128×128×128 с амплитудами, на выход — бинарная маска разломов того же размера.

Датасет: 200 объёмов для обучения, 20 для валидации, плюс реальный куб F3 Netherlands (512×384×128) для проверки на out-of-distribution данных.

## Предобработка

Каждый объём нормализуется отдельно: вычитаем среднее, делим на стандартное отклонение (z-score). Это обязательно, потому что амплитуды между синтетическими объёмами различаются, а на реальном F3-кубе диапазон вообще другой (от -32767 до 29793 против примерно [-2, 2] в синтетике).

Аугментации при обучении:
- Случайные отражения по всем 3 осям (p=0.5 каждая)
- Повороты на 90° вокруг вертикальной (depth) оси — это физически допустимо для сейсмики
- Случайное изменение яркости (±10%)
- Гауссовский шум (σ=0.05, p=0.3)
- 3D elastic deformation (p=0.3) — имитация деформаций пластов, добавлена начиная со второй модели

Повороты вокруг горизонтальных осей не использовались, т.к. в сейсмике вертикальная ось (время/глубина) имеет другой физический смысл.

## Модели

Обучены три модели разной архитектуры. Основная идея — ансамбль работает лучше, когда модели делают разные ошибки.

**BasicUNet v3 (13.87M параметров)**

Обычный 3D U-Net из библиотеки MONAI. Каналы: 32 → 64 → 128 → 256 → 256 → 32. Обучался 150 эпох, seed=42.

**BasicUNet-B (31.21M параметров)**

Расширенная версия того же U-Net. Каналы: 48 → 96 → 192 → 384 → 384 → 48. Обучался с другим seed=1337 и с elastic deformation в аугментациях. 100 эпох.

По сути это та же архитектура, только шире. Я надеялся что разные seed и аугментации дадут достаточно разнообразия для ансамбля, но на практике прирост оказался всего +0.005. Две почти одинаковые сети делают почти одинаковые ошибки.

**SegResNet (18.79M параметров)**

Архитектура на основе residual-блоков (тоже из MONAI). init_filters=32, blocks_down=(1,2,2,4), blocks_up=(1,1,1), InstanceNorm, dropout=0.2. 100 эпох, seed=2024.

Выбрана именно потому что принципиально отличается от BasicUNet: вместо обычных свёрток используются residual-блоки, другая структура декодера. Одиночный результат чуть ниже (0.7984 vs 0.8108), но ошибки другие — а это и нужно для ансамбля.

## Обучение

Общие параметры для всех трёх моделей:

- Loss: DiceCELoss — сумма Dice Loss и Binary Cross-Entropy с равными весами. Dice напрямую оптимизирует целевую метрику и устойчив к дисбалансу классов (разломы занимают ~3% объёма). BCE добавляет стабильность градиентов.
- Optimizer: AdamW, lr=2-3×10⁻⁴, weight_decay=10⁻⁵
- LR schedule: линейный warmup 5 эпох, потом cosine decay до 0
- AMP (fp16) для экономии памяти и ускорения
- Batch size: 2 (ограничение GPU)
- GPU: Kaggle T4 (16 GB)

Объёмы 128³ подаются целиком, без нарезки на патчи (они уже нужного размера).

## Работа с дисбалансом классов

Разломы занимают около 3% объёма — сильный дисбаланс. Использовал DiceCELoss, где Dice-компонента напрямую работает с соотношением пересечения и объединения, а не с отдельными вокселями. Это стандартный подход для медицинской/геологической сегментации с дисбалансом.

Пробовал ли другие loss: нет, DiceCELoss сразу дала хорошие результаты и менять не было необходимости. Focal Loss мог бы помочь, но при 3% дисбаланса (не 0.1%) DiceCELoss справляется.

## Test-Time Augmentation (TTA)

При инференсе каждый объём прогоняется через 8 вариантов: все комбинации отражений по трём осям. Каждый раз: отразить вход → предсказать → отразить обратно → усреднить вероятности.

TTA стабильно даёт +0.010-0.012 к Dice на всех моделях. Это бесплатный прирост — только время инференса увеличивается в 8 раз.

## Ансамбль

Финальное предсказание — взвешенное среднее sigmoid-вероятностей трёх моделей. Веса и порог бинаризации подбирались grid search на валидации.

Лучшая конфигурация: v3=1.0, B=1.0, SegResNet=0.5, порог=0.48.

SegResNet получает меньший вес, потому что одиночно слабее. Но ненулевой — значит его предсказания всё-таки добавляют информацию, которой нет у BasicUNet'ов.

## Результаты

| Модель | Dice (TTA) | IoU | Precision | Recall | Параметры |
|--------|-----------|-----|-----------|--------|-----------|
| BasicUNet v3 | 0.8107 | 0.6829 | 0.8018 | 0.8204 | 13.87M |
| BasicUNet-B | 0.8108 | 0.6831 | 0.8045 | 0.8178 | 31.21M |
| SegResNet | 0.7984 | 0.6656 | 0.7826 | 0.8158 | 18.79M |
| **Ансамбль** | **0.8147** | **0.6886** | **0.8030** | **0.8273** | — |

На F3: ансамбль даёт 2.37% fault fraction при пороге 0.48 (референс Wu et al.: 3.35%). Модели немного недо-предсказывают на реальных данных.

## Что пробовал и не сработало

**SwinUNETR (трансформер).** Val Dice 0.74, в 2 вариантах (с pretrained весами и без). На 200 синтетических объёмах трансформерам не хватает данных. В ансамбль не вошёл (оптимальный вес = 0).

**SWA (Stochastic Weight Averaging).** Взял обученный SegResNet, дообучил 12 эпох с циклическим LR, усреднил 3 snapshot'а. Результат: +0.000 к Dice. Модель уже была в плоском минимуме, усреднять нечего.

**2.5D подход (2D U-Net с ImageNet backbone).** EfficientNet-B0 + U-Net decoder, обучение на 2D срезах. Val 2D Dice 0.60 — плохо. ImageNet features не переносятся на сейсмику, слишком разные данные.

**Self-training на F3.** Хотел использовать согласованные предсказания ансамбля на F3 как pseudo-labels. Но консенсус трёх моделей дал только 0.7% positive вокселей (vs 3% в GT) — слишком мало и слишком смещено.

## Выводы

1. Архитектурное разнообразие — ключ к ансамблю. Две похожие сети (v3 и B) дают +0.005, добавление SegResNet ещё +0.001. Для серьёзного прироста нужны принципиально другие подходы.
2. TTA — самый надёжный бесплатный прирост (+0.01).
3. Per-volume z-score нормализация критична для переноса на реальные данные.
4. На 200 объёмах трансформеры проигрывают CNN.
